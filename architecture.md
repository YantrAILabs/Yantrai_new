# Architecture

## Overview

YantrAI is a static marketing website. It is built from plain HTML, CSS, JavaScript, and image assets with no build step or server-side application code.

## Components

- `index.html`: Main page markup and inline SVG sections.
- `styles.css`: Site styling and responsive layout.
- `script.js`: Small browser interactions, including reveal behavior, tabs, and the demo form UI.
- `assets/`: Static images used by the site.
- `app.yaml`: Google App Engine deployment configuration for static hosting.

## Hosting Flow

App Engine serves files directly from this repository:

```text
Browser request to appspot.com or custom domain
  -> App Engine static handler
  -> index.html, .html pages, CSS, JS, or asset files
```

There is no application server in this flow. App Engine only acts as a managed static file host.

## Deployment

Deploy with:

```bash
gcloud config set project yantraivisionos
gcloud app create --region=asia-south1
gcloud app deploy
```

The deployment uses the active Google Cloud project selected in the local `gcloud` CLI. App Engine app creation is a one-time setup step; the region cannot be changed after creation.

## Decisions

- App Engine was chosen because Firebase project initialization failed after the Google Cloud project was created.
- The site does not need Cloud Run or a container because it has no backend process.
- `app.yaml` only exposes common website file types so documentation and deployment files are not served as public assets.
- `automatic_scaling.max_instances` is set to `2` to make App Engine's scaling limit explicit for this low-traffic static site.

## Constraints

- Deployment requires the Google Cloud CLI.
- First deploys in new projects can require Storage Admin on the App Engine default service account so Cloud Build can use the staging bucket.
- App Engine's buildpack deployment can use Artifact Registry through `asia.gcr.io`; new projects may need `roles/artifactregistry.createOnPushWriter` on the App Engine default service account.
- Custom domains are mapped in App Engine and then pointed from GoDaddy DNS using the records generated by Google Cloud.
- App Engine issues managed SSL certificates for mapped custom domains, but activation can lag behind DNS propagation.
- Routes only work for files that exist in the deployed project.
- Links to pages that do not exist as `.html` files will return a 404.
